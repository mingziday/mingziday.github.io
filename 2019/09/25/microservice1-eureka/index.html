Spring Cloud微服务实践(1)：Eureka服务注册和发现

Eclipse创建父项目:microservice-parent

  1. pom类型：<packaging>pom</packaging>
  2. dependencyManagement 和 dependencies 的区别：
    dependencyManagement 用于声明子项目如果使用该依赖并且没有明确指定版本的情况下，使用该版本

Eclipse子项目：eureka-server

eureka-server为服务注册中心

  1. pom增加依赖：spring-cloud-starter-netflix-eureka-server
  2. yml 文件生成和配置
  3. 启动类EurekaServerApplication.java
    添加注解：
    @SpringBootApplication
    @EnableEurekaServer
  4. 启动

Eclipse子项目：eureka-client

eureka-client其实就是对外提供服务的具体应用，目前我还没想提供什么服务，就暂时取这个名字

  1. pom增加依赖：spring-cloud-starter-netflix-eureka-client
  2. yml 文件生成和配置
  3. 启动类EurekaClientApplication.java
    添加注解：
    @SpringBootApplication
    //@EnableEurekaClient 该注解在Spring Cloud Edgware以及更高版本中可以不使用
  4. 启动
  5. 访问 http://localhost:10001/，在nstances currently registered with Eureka栏目中可以看到注册到服务中心的微服务
