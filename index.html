Spring Cloud微服务实践(1)：Eureka服务注册和发现

Eclipse创建父项目:microservice-parent

  1. pom类型：<packaging>pom</packaging>
  2. dependencyManagement 和 dependencies 的区别：
    dependencyManagement 用于声明子项目如果使用该依赖并且没有明确指定版本的情况下，使用该版本

Eclipse子项目：eureka-server

eureka-server为服务注册中心

  1. pom增加依赖：spring-cloud-starter-netflix-eureka-server
  2. yml 文件生成和配置
  3. 启动类EurekaServerApplication.java
    添加注解：
    @SpringBootApplication
    @EnableEurekaServer
  4. 启动

Eclipse子项目：eureka-client

eureka-client其实就是对外提供服务的具体应用，目前我还没想提供什么服务，就暂时取这个名字

  1. pom增加依赖：spring-cloud-starter-netflix-eureka-client
  2. yml 文件生成和配置
  3. 启动类EurekaClientApplication.java
    添加注解：
    @SpringBootApplication
    //@EnableEurekaClient 该注解在Spring Cloud Edgware以及更高版本中可以不使用
  4. 启动
  5. 访问 http://localhost:10001/，在nstances currently registered with Eureka栏目中可以看到注册到服务中心的微服务

Java环境配置

JDK13安装

  1. 下载安装jdk-13_windows-x64_bin.exe
  2. 配置环境变量%JAVA_HOME%,maven需要这个环境变量

MAVEN安装

  1. 安装apache-maven-3.6.2-bin.zip
  2. 配置环境变量%MAVEN_HOME%
  3. 设置本地仓库地址：E:\repository
  4. 设置远程仓库阿里镜像：
    <mirror>
    <id>alimaven</id>
    <name>aliyun maven</name>
    <url>http://maven.aliyun.com/nexus/content/groups/public/</url>
    <mirrorOf>central</mirrorOf>
    </mirror>

Eclipse安装（必须要有JDK）

远程链接Linux的JDK是不行的，但是调试的时候应该可以

  1. 安装eclipse-inst-win64.exe

  2. 安装Spring插件

  3. 配置Eclipse的MAVEN参数
    settings文件路径，D:\service\apache-maven-3.6.2\conf\settings.xml

Hexo+GitHub，建立Github博客

一 Windows安装Hexo+GitHub，建立Github博客

参考：https://zhuanlan.zhihu.com/p/35668237
https://zhuanlan.zhihu.com/p/80140564

  1. 安装nodejs
  2. 安装Git
  3. 建立https://mingziday.github.io/
  4. 安装Hexo
    在E:\blogworkspace目录下打开git bash
    npm i hexo-cli -g 安装Hexo
    hexo init 初始化文件夹
    npm install安装必备的组件。
    npm i hexo-deployer-git 安装发布扩展
    hexo new post “article title” 新建一篇文章
    hexo g 生成静态网页
    hexo s
    打开本地服务器，浏览器预览 http://localhost:4000/
  5. 将本地的静态网页发布到GitHub
    如下步骤生产sshkey
    git config –global user.name “用户名”
    git config –global user.email “邮箱名@qq.com”
    ssh-keygen -t rsa -C “邮箱名@qq.com”
    在github头像下面点击settings，再点击SSH and GPG keys，新建一个SSH，title任意，内容为：
    cat ~/.ssh/id_rsa.pub
    hexo cl 清理
    hexo d 发布到github，第一次发布需要输入用户名密码
    在 https://mingziday.github.io/ 预览

二 博客美化小技巧

  1. 添加图片
    _config.yml里的psot_asset_folder:设置为true
    npm install hexo-asset-image –save ：再用命令生成md博客时，会建立一个目录用来存图片，这个插件版本为005才行
    注意图片文件的后缀，传到服务器自动变成了1.JPG

开发环境准备

硬件准备：

一个老的toshiba东芝笔记本，硬盘200G，内存2G，安装fedora服务器
一个新的Dell笔记本，硬盘200G，内存4G，安装windows 10家庭版，作为开发机

一 安装Fedora30

  1. 下载Fedora Server 30，地址：http://getfedora.org
  2. 用PowerIOS制作USB启动盘，用UltraISO制作的时候没有成功
  3. 从U盘安装Fedora，注意语言、分区、软件选择、网络配置
  4. 安装完成之后通过网络远程管理服务器：https://192.168.31.236:9090/system

二 安装图形界面

默认Fedora Server 是不安装图形界面的，renlevel=1，用如下命令安装GNOME
yum groups install “GNOME”
需要安装518个软件包，安装完成后启动：
startx
启动图形界面看起来还不错，以后应该没啥用

三 解决笔记本合上盖子后Fedora网络自动断开的问题

笔记本安装Fedora之后，之后都是通过ssh远程管理，要保证网络一直能通才行

  1. 修改配置文件/etc/systemd/logind.conf
    HandleLidSwitch=ignore
  2. 重启服务生效
    systemctl restart systemd-logind.service

四 配置Fedora SSH

  1. 检查ssh安装情况，默认已经安装
    rpm -qa | grep openssh-server
  2. 修改配置文件/etc/ssh/sshd_conf，打开下面内容的注释
    Port 22
    Protocol 2,1
  3. 重启ssh服务
    systemctl restart sshd
  4. 设置sshd开机启动（未设置）
    systemctl enable sshd
  5. 防火墙（内网访问，未设置）
    /etc/syscofig/iptables

-A INPUT -m state –state NEW -p tcp –dport 22 -j ACCEPT
systemctl restart iptables

五 Windows 安装SSH工具

  1. 下载Xmanager（未激活）
  2. 配置XShell，测试链接通过

六 Windows安装必备工具

  1. notepad++
  2. clover
  3. Wox
  4. everything
